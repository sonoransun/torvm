1. Introduction

1.1. 3D Map Projection

A new 3D projection map is available in Vidalia using the Marble Qt widget. If sufficient CPU power is available the responsiveness of this new 3D widget is easily usable.

2. Vidalia Marble Map Widget

2.1. Vidalia Marble Packages

There are two versions of the Vidalia Marble package. The first is a full Marbla data installer that is around double the usual size. If bandwidth or storage permits this is the recommended package type. The full installer can repair any of the individual generated tile images and eliminates initial run overhead for generating missing tiles on demand.

2.2. Launch Vidalia Marble

There is a link for the Marble and legacy versions of Vidalia. Be sure to launch the right Marble version of the controller.



   

If your system can support framerates in excess of 20 frames per second the Marble map should be pleasantly interactive. If host load or processing power is insufficient for the Marble map the usual 2D flat map widget should be used.



   

2.3. Minimal Marble Data Setup

The full set of image tiles uses a fair amount of space. If bandwidth is scarce the alternative minimal package can be used instead. This version will generate the tree of image tile files on demand.



   

The first time the minimal Marble version is run the image tiles will be generated on demand before Vidalia starts. If any of these files in the plugins data directory get corrupted you will need to remove the entire directory and re-create it again. The package repair option for data files is only available in the full Marble data package.
